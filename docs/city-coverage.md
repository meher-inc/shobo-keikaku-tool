# 対応消防本部カバレッジ

plan.todokede.jp（トドケデ消防計画）が専用様式に対応している消防本部の一覧と、
未対応エリアの扱いをまとめる。コードと乖離しやすいので、都市を追加・変更したら本表も更新する。

## ルーティングの仕組み

フォームの都道府県・市から TemplatePack を自動選択する。判定ロジックは1か所に集約している。

- 判定関数: [`lib/engine-v2/city-dispatch.ts`](../lib/engine-v2/city-dispatch.ts) `selectPackByLocation()`
- 回帰テスト: [`lib/engine-v2/tests/city-dispatch.test.ts`](../lib/engine-v2/tests/city-dispatch.test.ts)
- 呼び出し: [`app/api/generate-plan/route.ts`](../app/api/generate-plan/route.ts)（`?pack=` 明示指定がない場合）
- 画面の所轄ラベル: [`app/page.tsx`](../app/page.tsx) `deptName` / `deptKind`（**判定基準を上記と一致させること**）

判定方針:

- 政令市は **「市」単位** で判定する（画面ラベルと生成物を一致させるため）。
- **東京都のみ都道府県単位**（東京消防庁が都内をほぼ一円管轄するため）。
- 対応リスト外は `"full"`（京都市ベースの標準様式）へフォールバックする。

## 標準様式で対応する市

| 市 | 県 | 扱い |

| 浜松市 | 静岡県 | 公式の専用様式が Web 非掲載のため標準様式で作成。ただし所轄は判明しているため、画面では `deptKind = "named-standard"` で「浜松市消防局（標準様式で作成）」と表示する（`NAMED_STANDARD_DEPTS`）。 |
| 上記以外の対応外エリア | — | `"full"`（標準様式）。画面ラベルは「標準様式」。 |

岡山市は公式様式が記入式フォーム（表・テキストボックス）で体裁の逐語再現はできないが、
規定文（全19節）を逐語転記した内容忠実な散文版＋簡易ブランク表として実装した。

## 既知の修正（2026-06-16）

旧ルーティングは大阪府・神奈川県・福岡県・愛知県を **都道府県単位** で判定していたため、
対応市以外（例: 横須賀市・東大阪市）にもその県の代表都市の様式が適用され、
画面ラベルと生成物が食い違う不整合があった。`selectPackByLocation()` への集約に合わせ、
これらを「市」単位判定へ統一し、対応市以外は標準様式へフォールバックするよう是正した。
