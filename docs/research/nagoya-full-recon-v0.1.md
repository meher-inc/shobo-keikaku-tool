## 4. 「消防計画作成要領」との関係（共通基礎の解読）

`nagoya-yoryo.doc`（262.7 KB、1,552 行）は名古屋市消防局が定める **大規模事業所向けの全用途共通要綱**。「その他」「飲食店等」等の各用途別作成例は本要領のサブセット。

### 要領の章構造

| 章 | タイトル | 主な条 |
|---:|---|---|
| 第1章 | 総則 | 第1〜6条（目的、適用範囲、防火管理者等） |
| 第2章 | 予防管理対策 | 第7〜30条程度（予防管理組織、点検検査、火気管理、放火防止、自主検査） |
| 第3章 | 自衛消防活動 | （複数条） |
| 第4章 | 震災対策 | （複数条） |
| **第5章** | **警戒宣言発令時の応急対策** | 第31〜34条以降（警戒宣言時組織、東海地震注意情報、警戒宣言発令時の措置） |
| 第6章 | 防災教育、訓練 | （複数条） |

### 要領が定める「警戒宣言発令時の応急対策」（第5章、3節構成）

```
第１節　警戒宣言発令時の組織等
第２節　東海地震注意情報発表時から警戒宣言が発令されるまでの間における対応措置
第３節　警戒宣言発令から発生（又は解除）までの間における対策
```

**「その他」13条版での圧縮**: 要領第5章の3節構造を、第9条（注意情報〜警戒宣言）と第10条（警戒宣言発令時）の 2 条に圧縮した形。要領 第31〜34 条以降の本文を、中規模スコープに合わせて短縮。

### 要領の別表

要領内では別表1〜5まで言及あり（防火管理委員会、予防管理組織、自衛消防隊、警戒宣言発令時体制、夜間・休日体制）。「その他」は中規模スコープのため別表は委託状況票1本のみに簡略化。

## 5. 「大規模地震等ガイドライン」との関係

`nagoya-jishin-guideline.pdf`（8.5 MB、4,726 行）は **総務省消防庁発行（平成31年/2019年）の全国向けガイドライン**。名古屋市独自の文書ではない。

### ガイドライン構成

```
第１  消防計画の概要（消防計画とは / 対象災害 / 手続 / 内容 / 災害想定）
第２  具体的な消防計画の構成
        １ 総則的事項
        ２ 予防的事項
        ３ 応急対策的事項
        ４ 教育訓練
第３  超大規模防火対象物等に対する訓練の充実強化方策について
別添資料（複数）
```

### ガイドラインが「その他」ひな形に与える影響

- **「災害想定に基づいた消防計画の作成」**（第１節5）の方針が、第8条 (1)〜(4) の構成に反映（緊急地震速報・地震発生後の安全措置・自衛消防活動の段階別組み立て）
- **「対象となる災害」** が、第1条「火災、震災その他の災害」の文言根拠
- **「超大規模防火対象物」** （第3部）はガイドラインの応用範囲拡張で、「その他《中規模》」のスコープ外

ガイドラインは Phase 2 で実装する v2 テンプレに直接コピーするものではなく、設計判断の参照資料として扱う。

## 10. リスク・難所

1. **章立てなしテンプレを v2 ChapterSchema にマップする設計判断（Option A vs B）** — Option B 推奨だが、所轄消防署提出時に「公式テンプレと章立てが違う」と指摘されるリスクは osaka と同様に存在。提出実績の SHUN 確認推奨
2. **第8条 震災対策の長文構造（4小項+2 inline テーブル+各小項に詳細リスト）** — Section.body の単一配列で表現可能だが、テーブルは builders に切り出し必要。section の `body` と override の組み合わせが複雑になる可能性
3. **東海地震対応の法的根拠（大規模地震対策特別措置法）** — 名古屋市公式テキストの法令引用を一字一句保持必要。誤転記リスク高
4. **緊急地震速報・南海トラフ・東海地震の関係整理** — 「その他」では緊急地震速報（第8条 (2)）と東海地震注意情報（第9条）が別概念として扱われている。v2 placeholder/gating 設計でこの区別を明確に
5. **用途拡張時の共有モジュール設計** — その他/飲食店/ホテルで共有率 70%+ と推測されるが、実装は Phase 2A で「その他」のみ確定後、Phase 2B 着手時に共有切り出し
6. **本文中インラインテーブルの builders 化** — kyoto/tokyo/osaka/yokohama では別表（appendix）として扱うパターンだが、名古屋では条文本文の途中にテーブルが埋め込まれている。SectionOverride で条文 body の途中にテーブル emit を要する → buildSectionBody の章節制御が必要、kyoto/tokyo の override パターンとは異なる扱い
7. **作成要領（38+条、6章）と「その他」（13条フラット）の整合性** — 本実装は中規模スコープのため要領の中規模相当を採用。大規模事業所向けに要領全文を実装する必要が将来出た場合、別 pack（`nagoya-city.full-large.json`）として隔離
