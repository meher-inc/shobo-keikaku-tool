## PoCの目的と成功基準

### 目的

月額サブスク「プロプラン¥19,800」の差別化機能として、**AIによる消防計画の自動ドラフト生成**を実装する。従来のウィザード入力方式に対して、**10問程度のヒアリングで消防計画の8割が完成する**体験を作る。

### PoCで検証したい3つの問い

1. **品質**：Claude APIで生成した消防計画は、人間が書くレベルに到達できるか
2. **速度**：ヒアリング→生成→出力までの体感時間は5分以内に収まるか
3. **ロバスト性**：業種・規模のバリエーション（10パターン程度）で破綻しないか

### 成功基準（Day 60 = 6/11時点）

- [ ] 消防計画の主要5章（総則、予防管理、自衛消防、避難、訓練）が自動生成される
- [ ] 3種類の業種（飲食店/介護施設/オフィス）で動作確認済み
- [ ] 元消防士視点で「これは使える」と判断できる品質
- [ ] 生成時間が1分以内
- [ ] 人間レビュー画面で修正可能な構造になっている

## 既存プロダクトとの位置付け

### 現状のウィザード方式（継続）
- 20-30問の項目入力 → テンプレート埋め込み
- 網羅性は高いが、入力負荷も高い
- 月額ミニマム・スタンダードプランで継続利用

### AI自動生成方式（新規追加）
- 10問程度のヒアリング → AI生成 → 人間レビュー
- 入力負荷を半減、体験を劇的に改善
- 月額**プロプラン限定機能**として差別化
- 既存のdocx-jsテンプレートエンジンと接続

### 両立の設計思想
AI生成は**ウィザード方式を置き換えない**。プロプランユーザーは「AI生成でドラフト作成 → ウィザード画面で微調整」という2段階フローが可能になる設計。

## 入力設計：ヒアリング10-15問

### Tier 1：必須項目（Q1-Q7）

事業所の基本情報を取得する7問。これだけで60%の消防計画生成が可能。

**Q1. 事業所の名称と所在地**
- 入力形式：フリーテキスト
- 用途：消防計画の冒頭「対象事業所」欄

**Q2. 事業の業種は次のどれに該当しますか？**
- 入力形式：選択肢
  - 飲食店・物販店
  - 介護施設（特養・デイサービス等）
  - 保育園・幼稚園
  - オフィスビル・事業所
  - 共同住宅
  - 学校
  - 病院・診療所
  - 工場・倉庫
  - その他（自由記述）
- 用途：業種別の予防管理基準の適用

**Q3. 建物の構造は？**
- 入力形式：選択肢
  - 耐火構造（RC造・SRC造）
  - 準耐火構造
  - 木造
  - その他
- 用途：避難経路・初期消火の方針決定

**Q4. 階数と延床面積は？**
- 入力形式：数値入力
  - 地上___階 / 地下___階
  - 延床面積___㎡
- 用途：収容人員・防火管理者の要否判定

**Q5. 従業員数と最大収容人員は？**
- 入力形式：数値入力
- 用途：自衛消防組織の編成、訓練計画の規模決定

**Q6. 管轄消防署は？**
- 入力形式：選択肢（京都市中京消防署 等、主要署をリスト化）
- 用途：届出先、管轄固有の様式・要求事項の適用

**Q7. 防火管理者の氏名と資格区分は？**
- 入力形式：フリーテキスト＋選択肢
  - 氏名___
  - 資格：甲種 / 乙種
- 用途：消防計画の選任届欄

### Tier 2：詳細項目（Q8-Q12）

より精度の高い生成に必要な5問。

**Q8. 主要な出火危険源は？（複数選択）**
- 選択肢：厨房火器 / 電気設備 / 喫煙場所 / 暖房器具 / 危険物 / その他
- 用途：予防管理の重点項目

**Q9. 消防設備の設置状況は？（複数選択）**
- 選択肢：屋内消火栓 / スプリンクラー / 自動火災報知設備 / 誘導灯 / 避難器具 / その他
- 用途：自衛消防活動の構成

**Q10. 営業時間・勤務時間帯は？**
- 入力形式：時間入力
- 用途：夜間・休日の対応体制

**Q11. 特殊な配慮が必要な利用者・入居者はいますか？**
- 選択肢：高齢者 / 乳幼児 / 身体障害者 / 外国人 / なし
- 用途：避難誘導計画の具体化

**Q12. 過去の災害・事故・訓練実績は？（任意）**
- 入力形式：フリーテキスト
- 用途：教訓反映・継続改善の記述

### Tier 3：オプション項目（Q13-Q15）

品質を上げるための追加項目。スキップ可能。

**Q13. 近隣との相互応援協定はありますか？**
**Q14. 危険物取扱責任者の選任状況は？**（危険物関連業種のみ）
**Q15. 特記事項や懸念事項があれば記述**

## 出力設計：消防計画の構造

### 生成する章立て（消防法施行規則第3条準拠）

```
第1章 総則
  1. 目的
  2. 適用範囲
  3. 管理権原者の責務
  4. 防火管理者の責務

第2章 予防管理
  1. 防火対象物の日常点検
  2. 火気使用設備・器具の点検
  3. 避難施設の管理
  4. 収容人員の管理
  5. 定期点検（消防設備・防火設備）

第3章 自衛消防の組織
  1. 自衛消防組織の編成
  2. 各班の任務
  3. 非常招集体制

第4章 火災発生時の対応
  1. 通報連絡
  2. 初期消火
  3. 避難誘導
  4. 応急救護

第5章 避難計画
  1. 避難経路
  2. 避難場所
  3. 特別な配慮が必要な者の避難
  4. 避難確保計画（該当業種のみ）

第6章 教育・訓練
  1. 消防訓練の実施計画
  2. 防災教育の実施
  3. 訓練記録

第7章 その他
  1. 消防機関との連絡
  2. 計画の見直し
  3. 付属資料
```

### 出力フォーマット

**Phase 1（Week 4-6）**: JSON構造化出力

**Phase 2（Week 7-10）**: 既存docx-jsテンプレートエンジンと接続し、Word文書として出力

## プロンプト設計（初期案）

### システムプロンプト

```
あなたは日本の消防法規に精通した、元消防士の防災コンサルタントです。
以下の事業所情報に基づき、消防法施行規則第3条に準拠した消防計画を作成してください。

作成にあたっての原則：
1. 実務上実行可能な内容にすること（理想論ではなく現場で回る内容）
2. 業種特性を反映すること（介護施設と飲食店では避難計画が大きく異なる）
3. 管轄消防署の様式・要求水準を意識すること
4. 曖昧な表現を避け、具体的な手順・担当者・時刻を記述すること
5. 法令の誤った解釈や、古い法令に基づく記述を避けること

出力は指定されたJSON構造で返してください。
```

### 品質向上のための工夫

1. **業種別のFew-Shot Examples**
   - 業種ごとに3-5個の良い例を事前に用意
   - プロンプトに「この業種ではこう書く」例を動的挿入

2. **管轄別の様式バリエーション**
   - 京都・東京・大阪など主要都市の様式差異を反映
   - 管轄選択時に対応するプロンプト修飾

3. **チェーン・オブ・ソート（Thinking Mode）**
   - Claude Sonnet 4.6 の拡張思考で、章ごとに論理整合性を確認
   - 特に業種固有のリスク識別で効果大

4. **Prompt Caching**
   - システムプロンプト部分をキャッシュして90%コスト削減
   - 同一ユーザーが複数回生成する場合に効果大

## 技術スタック

### 選定ツール

| 層 | 採用 | 理由 |
|---|---|---|
| LLM | Claude Sonnet 4.6 | 日本語品質最高、長文生成安定、コスト効率 |
| API利用 | Anthropic API | 既存プロジェクトで実績 |
| バックエンド | 既存スタック踏襲 | 新規導入でリスク増やさない |
| テンプレエンジン | 既存docx-js | 資産活用 |
| フロントエンド | 既存React | ウィザードのUIコンポーネント流用 |
| 認証 | 既存の認証基盤 | プロプラン判定はここで |

### API設計

```
POST /api/fire-plan/generate
Body: {
  hearing_answers: {...},
  plan_type: "standard",
}
Response: {
  plan_id: "uuid",
  status: "generating"
}

GET /api/fire-plan/{plan_id}
Response: {
  status: "completed",
  content: { ... }
}
```

### コスト試算

- 1回の生成あたり：入力2,000トークン + 出力12,000トークン
- Claude Sonnet 4.6 料金で1回約30-50円
- プロプラン ¥19,800/月 × 月3回生成 = 月150円のAPIコスト
- 粗利率ベースで問題なし（利益率80%超維持可能）

## 評価基準（Day 60判定用）

### 品質評価（5段階）

| 評価 | 基準 |
|---|---|
| 5 | そのまま消防署に提出可能 |
| 4 | 軽微な修正で提出可能（実使用レベル） |
| 3 | 章単位で数箇所の修正必要（叩き台として有用） |
| 2 | 構造はあるが、内容の大幅書き換えが必要 |
| 1 | 業種や法令の理解が誤っており、使えない |

**目標：全10パターンで平均4以上**

### 10パターンの検証ケース

| # | 業種 | 規模 | 構造 |
|:---:|---|---|---|
| 1 | 飲食店（居酒屋） | 小規模（30席） | 木造2階 |
| 2 | 飲食店（チェーン） | 中規模（100席） | RC造1階 |
| 3 | 介護（通所） | 50名定員 | RC造3階 |
| 4 | 介護（特養） | 100床 | SRC造4階 |
| 5 | 保育園 | 80名 | RC造2階 |
| 6 | オフィス | 50名 | RC造10階の3フロア |
| 7 | 共同住宅 | 20戸 | RC造5階 |
| 8 | 病院（診療所） | 10床 | RC造2階 |
| 9 | 物販店 | 500㎡ | 鉄骨造1階 |
| 10 | 工場 | 30名 | 鉄骨造2階 |

### 速度評価

- 初回生成：90秒以内
- 再生成（修正依頼）：60秒以内
- ヒアリング入力：3分以内

### ロバスト性評価

- 入力が部分的に不完全でもエラーで止まらない
- 業種選択が「その他」でも最低限の計画を生成
- 特殊業種（パチンコ店・ガソリンスタンド等）でも対応 or 適切なエラー

## リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| 生成品質が期待水準に届かない | 中 | プロプラン差別化機能を別のものに振り替え（複数事業所管理を主軸に） |
| 管轄消防署ごとの様式差異に対応できない | 中 | 京都・東京のみ優先対応、他地域は後回し |
| Claude APIのレート制限 | 低 | バックオフ実装、エラー時の再試行 |
| 法令改正への追従 | 中 | システムプロンプトに最新法令リンクを組み込み、定期更新 |
| 誤った消防計画を生成して事故に繋がる | 高 | 「AI生成後は必ず人間レビューを」の明示的警告をUI上に表示 |
